Turning bug data into battle-ready priorities.

BUG DATA STRUCTURE
type Bug struct {

  ID                string    // Unique identifier
  Title             string    // Bug title
  Severity          int       // Severity (1-10)
  Age               int       // Age in days
  BountyValue       float64   // Reward amount (USD)
  Reproductions     int       // How easy to reproduce
  EstimatedFixHours float64   // Estimated hours to fix
  Source            string    // Where it was found
  URL               string    // Link to bug report
  Priority          float64   // Calculated score (0-100)

}

Core Features

Four questions, answered instantly.

⚡

Priority Scoring

Weighted formula across severity, bounty, reproductions, and age — normalized to 0-100.

🏆

Max-Heap Queue

Top-K retrieval in O(log n) via Go's container/heap. Instant access to what matters most.

🔄

Fix Simulation

Deploy a fix → heap rebalances → priorities shift. See the impact of every decision live.

📅

Greedy Scheduler

Maximize total priority within developer-hour budgets. Like a knapsack solver for bugs.

🧠

AI Explanations

Optional layer explains why a bug ranks highly. Falls back to deterministic templates.

📊

Greedy vs Optimal

Brute-force comparison validates greedy — see exactly how close it gets to optimal.
